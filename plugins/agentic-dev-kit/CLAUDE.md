# Agentic Dev Kit

Multi-agent system for structured software delivery. Three cells, explicit governance, explicit handoffs.

---

## Session start

1. Check `memory/progress.md` — if a task is in progress, load it before anything else
2. Check `REPOSITORY-CONTEXT.md` in the project root — load it if present
3. If `REPOSITORY-CONTEXT.md` is missing, tell the user to run `/setup` before starting any delivery work

---

## Agent registry

### Development Cell

| Agent | `name:` | Invoke for |
|---|---|---|
| Orchestrator | `development-orchestrator` | Feature work, code changes, spec, implementation, review |
| Context Mapper | `context-mapper` | Mapping affected files, deps, risks before changes |
| Design Analyst | `design-analyst` | Option exploration (explore) or pattern analysis (ground) |
| Architecture Challenger | `architecture-challenger` | Challenge pass — drift, coupling, stronger alternatives |
| Migration Planner | `migration-planner` | Schema changes, breaking API changes, multi-phase rollouts |
| UI Designer | `ui-designer` | Screen structure, flow, design-system alignment |
| UI Reviewer | `ui-reviewer` | UI critique — consistency, reuse, accessibility |
| Spec Writer | `spec-writer` | Execution contracts written into worktrees |
| Implementer | `implementer` | Code + tests, exactly to spec |
| Reviewer | `reviewer` | Independent correctness and spec compliance review |
| Security Reviewer | `security-reviewer` | Application-level security review of diffs |

### Delivery Engineering Cell

| Agent | `name:` | Invoke for |
|---|---|---|
| Orchestrator | `delivery-orchestrator` | Deployment readiness reviews |
| Pipeline Engineer | `pipeline-engineer` | CI/CD, build flow, rollback readiness |
| Platform Evaluator | `platform-evaluator` | Runtime fit, hosting, scaling, cost |
| DevSecOps Reviewer | `devsecops-reviewer` | Secrets, IAM/RBAC, supply chain |
| Observability Engineer | `observability-engineer` | Logging, metrics, tracing, alerting |

### QA Cell

| Agent | `name:` | Invoke for |
|---|---|---|
| Orchestrator | `qa-orchestrator` | Scope E2E tests from spec, drive QA Engineer, synthesise report |
| QA Engineer | `qa-engineer` | Author + run Playwright tests headlessly |

### Shared

| Agent | `name:` | Invoke for |
|---|---|---|
| Context Mapper | `context-mapper` | Used by both Development and Delivery cells |
| Docs Updater | `docs-updater` | Documentation and repository memory updates |

## Cell routing

| Task type | Start here |
|---|---|
| Feature work, code change, design, spec, implementation, review | `development-orchestrator` |
| CI/CD, deployment, runtime, platform, observability, cost | `delivery-orchestrator` |
| E2E acceptance testing against a deployed test environment | `qa-orchestrator` |

When in doubt, start with `development-orchestrator`. It hands off explicitly.

---

## Slash commands

| Command | Purpose |
|---|---|
| `/setup` | Configure the kit for your project stack |
| `/dev` | Invoke the Development Cell Orchestrator |
| `/deliver` | Invoke the Delivery Engineering Cell Orchestrator |
| `/qa` | Invoke the QA Cell Orchestrator |
| `/handoff` | Generate a cross-cell handoff document |
| `/status` | Show current task progress |

## Context files (plugin-owned)

These live in the plugin and are loaded via `${CLAUDE_PLUGIN_ROOT}/context/`:

- `${CLAUDE_PLUGIN_ROOT}/context/instructions.md` — universal engineering rules
- `${CLAUDE_PLUGIN_ROOT}/context/development-charter.md` — Development Cell governance
- `${CLAUDE_PLUGIN_ROOT}/context/delivery-charter.md` — Delivery Engineering Cell governance
- `${CLAUDE_PLUGIN_ROOT}/context/qa-charter.md` — QA Cell governance

## Project-generated files (live in user's project)

- `REPOSITORY-CONTEXT.md` — generated by `/setup`; project-specific facts, commands, structure
- `.claude/instructions-stack.md` — generated by `/setup`; stack-specific engineering rules
